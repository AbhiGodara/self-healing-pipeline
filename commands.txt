=================================
IMPORTANT PIPELINE COMMAND CHEATSHEET
=================================

=== API / BACKEND COMMANDS ===
Start the FastAPI server (Run this in your pipeline root directory, with your SHPenv virtual env activated):
> uvicorn api.main:app --reload

=== DOCKER COMPOSE COMMANDS ===
Start Background Infrastructure (Postgres, Airflow):
> docker-compose up -d

Stop Background Infrastructure:
> docker-compose down

Rebuild Infrastructure completely from scratch (destroys old data):
> docker-compose down -v
> docker-compose up -d --build

=== DB INTERACTION (PSQL) ===
Access the main pipeline database manually through the Docker container:
> docker exec -it self_healing_pipeline-postgres-1 psql -U pipeline_user -d pipeline_db

-- Important SQL Queries to run inside PSQL:
Show the main sales tracking table:
=> SELECT * FROM raw_sales;

Show exactly what data the AI safely deleted and quarantined:
=> SELECT * FROM quarantine_sales;

Show the AI Agent's thought process tracking log:
=> SELECT agent_name, action_taken, success, created_at FROM audit_log;

Check exactly which tables currently exist:
=> \dt

Exit PSQL:
=> \q

=== AIRFLOW CLI COMMANDS ===
List the last 10 pipeline executions directly from the CLI to ensure it hasn't crashed:
> docker exec self_healing_pipeline-airflow-webserver-1 airflow dags list-runs -d data_ingestion_pipeline

Manually Trigger the DAG from backend:
> docker exec self_healing_pipeline-airflow-webserver-1 airflow dags trigger data_ingestion_pipeline

=== ENVIRONMENT SETUP ===
To activate your virtual environment (Windows Powershell):
> .\enviorenments\SHPenv\Scripts\Activate.ps1
